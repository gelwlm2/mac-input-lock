## Mac 输入法为什么会自动切换？

macOS 系统内置了一个叫"自动切换到文稿的输入法"的功能。本意是好的——让每个 App 记住自己上次用的输入法状态，切过去就自动恢复。

问题是，这个逻辑并不总是那么聪明。

有时候密码输入框会强制切到 ABC，有时候某个弹窗 App 会偷偷帮你换掉输入法，有时候你根本不知道是哪个操作触发了切换。结果就是——你以为自己在中文状态，打出来的是英文字母；你以为自己锁定了英文，突然冒出一个汉字。

> **一句话定义**：Mac 输入法锁定，是指通过工具强制保持某一个输入法始终处于激活状态，无论切换多少个 App、触发多少次系统事件，都不会被意外替换。

系统设置里关掉"自动切换"这个选项，能解决一部分问题，但不是全部——有些 App 会绕过这个设置，直接切换输入法。这就是 InputLock 存在的原因。

## 它不能做什么？

把这个说清楚很重要。

InputLock 是一个"锁定"工具，不是"按 App 自动切换"工具。如果你的需求是"打开 Xcode 自动切英文，打开微信自动切中文"，这种按 App 分配输入法的功能，InputLock 并不提供——那是 KeyboardHolder 或 SwitchKey 之类工具的定位。

InputLock 做的是另一件事：**你选定一个输入法，它帮你死守住，无论切什么 App 都不让别人动它**。

这两种需求看起来相似，但逻辑不同。想清楚自己要的是哪种，再做决定。
